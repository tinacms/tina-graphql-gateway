Interface Builder

The builder holds all the functions which are required to build the schema, everything starts with the documentUnion, which then trickles down through the schema, populating all the fields by reading the settings.yml and template definition files.

documentTaggedUnionInputObject

documentTaggedUnionInputObject: (args: { cache: Cache; section?: undefined | string }) => Promise<GraphQLInputObjectType>
  • Defined in packages/gql/src/builder/index.ts:214

The input values for mutations to the document from a section

# Example
input DocumentInput {
 PostInput: PostInput
 AuthorInput: AuthorInput
}

Note that while this isn't a union, it's behaving close to one, GraphQL doesn't support unions in mutations, so instead each key (ex. PostInput) is the property we require when accepting a mutation.

So if your payload looked like this it'd be considered invalid, while it's possible to provide both PostInput and AuthorInput from the schema's perspective , we'll throw an error, changing the author data at path posts/1.md makes no sense:

{
 "path": "posts/1.md",
 "params": {
   "PostInput": {
     "data": {
       ...
   "AuthorInput": {
     "data": {
       ...

Read more about the trade-offs here

documentUnion

documentUnion: (args: { cache: Cache; section?: undefined | string }) => Promise<GraphQLUnionType>
  • Defined in packages/gql/src/builder/index.ts:248

The top-level result, a document is a file which may be of any one of the templates defined, the union consists of each template which is possible.

union DocumentUnion = Post | Page

Note that this is also used on section-level references as well. If you have a post template which has an authors reference, that author document could consist of multiple template types. An example might be that some authors use a more detailed template, while others are very basic

union authorsDocumentUnion = DetailedAuthor | BasicAuthor

In this example the authorsDocumentUnion would be referenced in the Post's type (notice that the authorsDocumentUnion is "namespaced" by the section slug authors)

type PostData {
  title: String
  author: AuthorDocument
}

type AuthorDocument {
  document: authorsDocumentUnion
}

union authorsDocumentUnion = DetailedAuthor | BasicAuthor
